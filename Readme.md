# Projektstruktur #

Ein Projekt besteht aus mehr als nur die Quellen. Alle verschiedene Artefakte sollen ihren Platz finden.

* 01 Specifications 
  * Ideas, Concepts (Brainstorming-Protokolle, Architekturscribbles und Ähnliches) 
  * Requirements (Anforderungsdokumente)
  * Specifications
* 02 Design
  * Code & Architecture
  * UI
* 03 Code
  * Help Projects
  * Installer Projects
  * Prototypes (Kleine Projekte, um etwas auszuprobieren)
  * Source 
  * Tools (Projekte, um die Quellen zu bauen)
* 04 Review
  * Code
  * Design
* 05 Test
  * Integration Testing
  * Unit Testing
* 06 Documentation
  * Legal Regulations (U.a. alles was mit den Lizenzen der verwendeten Bibliotheken zu tun hat)
  * Technical Documentation
  * User Documentation 
* 07 Sample Data
* 10 Project Management
  * Meetings


Die Ordner *01 Specifications\Specifications* und *01 Specifications\Requirements* sollten in der Regel leer sein, da das Backlog im ALM gepflegt wird.
